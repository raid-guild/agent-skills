# Source To Draft

Use this when turning raw notes, memory, digests, workflows, Portal records, or user context into content.

## Source Ledger

Separate:

- confirmed facts
- inferred implications
- missing facts
- private/internal details
- public-safe receipts
- approval status

## Draft Brief Shape

```text
Audience:
Channel:
Content job:
Primary source facts:
Angle:
What not to claim:
Missing facts:
Voice layer:
Recommended format:
Review gates:
```

## Rules

- Do not make claims more specific than the source allows.
- Preserve uncertainty: draft, proposed, in review, planned, blocked, waiting on approval.
- Use live Prism Memory for recency when needed, but do not copy private memory into public text.
- Prefer public receipts for public claims.
- If source material is thin, produce a brief and missing-facts list before drafting.

## Hand Off

After strategy:

- use `rg-brand-voice` for voice/persona polish
- use `rg-public-output-safety` before public output is posted, queued, or marked ready
