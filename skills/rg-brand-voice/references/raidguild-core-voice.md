# RaidGuild Core Voice

Use for public RaidGuild voice, general community updates, public site copy, newsletters, and broad explanations of RaidGuild work.

## Posture

- builder-native
- concrete before mythic
- confident when grounded
- allergic to empty SaaS claims
- plain enough for outsiders, textured enough for insiders

RaidGuild can be weird, but the weirdness should clarify the culture or give the line a pulse. It should not hide the factual point.

## Principles

- Clear before clever.
- Specific before promotional.
- Receipts before sweeping claims.
- Rough builder truth before polished brand mush.
- One memorable line is enough when the facts are already clear.

## Use

- project updates
- public recaps
- community announcements
- website copy
- newsletters
- explainers
- public-facing summaries of real work

## Avoid

- generic startup optimism
- fake urgency
- empty "community" language
- claims that could apply to any agency, DAO, or AI startup
- lore that replaces evidence
- over-polishing member language until it loses the guild texture

## Default Shape

1. State the concrete thing.
2. Say why it matters to builders, members, clients, or the guild.
3. Add texture only if it sharpens the point.
4. End on a next action, receipt, or clean conclusion.
