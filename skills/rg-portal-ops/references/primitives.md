# Portal Primitives

Use the Portal as the presentation and coordination layer for real community activity.

Use repo-level `references/portal-cms-model.md` for field-level detail when needed.

## Core Collections

- `activityItems`: factual dated signals; what happened.
- `threads`: ongoing lines of work or thought; what keeps evolving.
- `events`: sessions and calendar anchors; when people should show up.
- `projects`: live collaboration surfaces; what is being built.
- `dailyBriefs`: assembled current snapshots; what matters now.
- `profiles`: people and contributors; who is involved.
- `spotlights`: admin/editorial placement for what should be front and center.
- `wikiPages`: durable source-backed topic pages; what the community has learned.
- `posts`: reviewed editorial or distribution artifacts.
- `pageCopy`: CMS-managed copy for fixed product-flow pages.

## Source Rules

- Do not invent activity, people, dates, links, claims, or decisions.
- Preserve uncertainty when source inputs are incomplete.
- Prefer updating existing continuity records over creating duplicates.
- Create new records only when the source introduces a distinct real object.

## Agent Account Flow

Agent accounts are trusted automation identities. Create or use them only when target environment and credentials are provided.

Agents must not:

- delete records without explicit confirmation
- manage users
- impersonate humans
- publish sensitive content without approval
