# Evidence And Announcements

Use after Bankr actions or when preparing public communication.

## Evidence To Capture

When available, capture:

- command run
- account/wallet context, without secrets
- chain/network
- token address
- tx hash
- Bankr URL
- token page
- fee claim receipt
- failure summary

Do not store:

- private keys
- seed phrases
- raw auth links
- cookies
- API keys
- verbose logs containing secrets

## Public Output

Before public announcements:

1. Confirm the action actually happened from command/API evidence.
2. Confirm the public details are safe to disclose.
3. Use `rg-public-output-safety`.
4. Use `rg-brand-voice` or `rg-content-strategy` if copy needs shaping.
5. Use `rg-publishing-ops` for sending, scheduling, or URL reconciliation.

Do not announce a token launch, fee claim, trade, or wallet action based only on an intended command.
