# Draft

Produce a complete blog post draft with source notes grounded in Prism Memory and Prism Knowledge context. Save the draft as a request artifact and return a short summary of the created draft artifact plus any factual risks the reviewer should check.

When it materially improves the piece, pull a few short notable quotes from previous meetings through Prism Memory and weave them into the draft. Prefer quotes that sharpen the core argument, illustrate community voice, or make a concrete point more memorable.

When using quotes:

- keep them short and selective
- cite the meeting artifact id, date, or source note in the draft's source-notes section
- do not invent wording; if exact wording is uncertain, paraphrase instead of quoting
- prefer recent meeting summaries or artifacts over vague memory-only references
