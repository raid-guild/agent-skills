# Triage

## Outcome

Decide whether this request is ready to become a Queen Raida site article, and record useful implementation notes for the Work step.

## Required Context

Use the linked change request, operator comments, artifacts, external refs, and the Queen Raida target app metadata. The request must be scoped to https://github.com/raid-guild/Queen-Raida. If it is not, say so and recommend retargeting or using another workflow.

Review the current repository guide at site/AGENTS.md before recommending implementation details. The guide is authoritative for article file location, frontmatter, markdown format, validation commands, and safety rules.

## Checks

Confirm the request is for creating a specific Queen Raida website article, not a broad site redesign or unrelated code change.

Confirm there is enough public or approved source material to write the article. Identify any missing facts, links, quotes, dates, names, or claims that need operator confirmation.

Confirm the expected article angle, intended audience, likely slug, and any public receipts that should be linked.

Screen for privacy and publication risk. Do not recommend publishing private Discord content, private client work, private member details, secrets, internal URLs, unsupported claims, or operational debug language.

## Artifacts And Request Updates

Create or update a markdown artifact named triage-fix-notes.md with:

- Article goal and scope
- Source material and public receipts to use
- Proposed filename or slug, if clear
- Safety notes and known exclusions
- Suggested implementation steps for Work
- Open questions, if any

Patch the request triage summary and agent recommendation with concise, actionable detail.

## Completion

If ready, recommend moving to the Approve gate. If not ready, clearly state what information is missing before approval.
