# Media Plan

Create `media-plan.md`.

Determine whether the post needs:

- hero image
- inline image
- social image
- screenshots
- inline SVG/chart/diagram
- no media

Rules:

- Recommend generated images only when they materially help the post.
- Limit inline image proposals to at most two unless the operator explicitly requests more.
- Provide alt text and placement hints for selected media.
- Avoid misleading visuals, unsupported depictions, logos, readable text in generated images, and private/person-identifiable depictions.
- For `fireside`, a generated header image is usually expected unless the operator approves a fallback.
- For `lore`, image direction should support continuity and story mood without implying unsupported real-world claims.

If image generation is needed, produce prompts suitable for `imagegen` and record them in the plan. The workflow may create media in a separate image generation step in runtimes that support it.
