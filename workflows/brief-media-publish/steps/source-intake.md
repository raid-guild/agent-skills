# Source Intake

Use `rg-content-strategy` and `rg-portal-ops` when Portal records or session artifacts are involved.

Read the approved steering note, request description, attached artifacts, external refs, Portal records, Prism Memory digest, and source URLs.

Produce `brief-source-ledger.md` with:

- source mode and visibility
- time window or session id
- primary source list with URLs, artifact ids, or record ids
- public-safe facts
- internal-only facts, if any
- unsupported claims and gaps
- candidate highlights
- required follow-up questions only when the workflow cannot proceed safely

For public outputs, separate direct evidence from interpretation.
