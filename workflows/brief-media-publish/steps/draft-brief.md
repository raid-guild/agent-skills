# Draft Brief

Use `rg-brand-voice`, `rg-content-strategy`, and `rg-public-output-safety` for public outputs.

Draft the written brief from `brief-source-ledger.md` and `brief-plan.md`.

Requirements:

- keep claims grounded in the source ledger
- preserve visibility boundaries
- make public versions understandable without private context
- avoid debug/process language and unsupported hype
- include useful links when approved sources provide them

Produce `brief-draft.md` with:

- title
- summary
- sections or bullets appropriate to the mode
- link list
- public safety notes, if public
- internal-only notes, if internal
