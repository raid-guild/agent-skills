# Source Intake

Use `rg-content-strategy`, `rg-brand-voice`, and `rg-portal-ops` when Portal context is relevant.

Read the approved steering note, request description, artifacts, external refs, and source URLs. If the request references a Prism Memory item, X collection summary, Portal item, or campaign state artifact, fetch or summarize that source before broader research.

Produce `queen-raida-social-intake.md` with:

- source mode and target action
- source URLs, item ids, or artifact ids
- factual claims that are directly supported
- gaps, unsupported claims, and items needing human confirmation
- item classification: `support_request`, `reply_opportunity`, `cohort_signal`, `agency_signal`, `project_signal`, `member_props_opportunity`, `campaign_beat`, `announcement`, `spam_or_low_value`, or `needs_human_review`
- seriousness level: `normal`, `sensitive`, `support`, or `high_risk`
- recommended response mode: `reply`, `quote`, `new_post`, `thread`, `queue_only`, `no_action`, or `needs_more_context`

Do not draft public copy in this step.
