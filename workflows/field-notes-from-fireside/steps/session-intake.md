# Session Intake

Use `rg-portal-ops` and `rg-content-strategy`.

Read the approved steering note, request description, Portal session/event record, attached artifacts, external refs, and source URLs.

Produce `fireside-session-intake.md` with:

- Portal session/event id and URL
- available transcript, summary, recording, and source artifacts
- missing artifacts
- desired child outputs
- known people, projects, tools, themes, and constraints
- blockers that prevent source-pack work

Do not infer missing session facts. Mark unknowns explicitly.
